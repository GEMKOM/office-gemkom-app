Route Protection Test

This page tests the route protection system to ensure users cannot access URLs they don't have permission for.

Test 1: Check Current User Access

This will show what routes the current user has access to based on their team.

Test 2: Test Route Protection

Try to access different routes to see if protection is working.

Test 3: Simulate Direct URL Access

This simulates what happens when a user tries to access a URL directly.

Instructions

  1. Make sure you are logged in to the application
  2. Click "Check My Access" to see what routes you have access to
  3. Try accessing different routes to see if protection works
  4. Try accessing a URL directly in the browser that you shouldn't have access to
